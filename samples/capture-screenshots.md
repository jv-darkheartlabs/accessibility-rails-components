# Screenshot capture guide

This repo is set up to support consistent, portfolio-friendly screenshots in GitHub README files.

## Where to put images
- Store screenshots in `assets/screenshots/`
- Keep filenames stable so README links don't break

## Naming convention (recommended)
- `assets/screenshots/01-hero.png`
- `assets/screenshots/02-component-preview-grid.png`
- `assets/screenshots/03-keyboard-navigation.png`
- `assets/screenshots/04-accessibility-audit.png`

## Recommended shots for this repo
1. **Component preview grid** (`/rails/view_components`)
2. **Keyboard-only navigation** (highlight tab order / focus states)
3. **Screen reader output** (terminal or VoiceOver caption mode)
4. **Accessibility audit** (axe CLI or DevTools report)

## Image format
- Prefer PNG
- Target ~1200px wide
